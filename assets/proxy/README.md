# Proxy Assets

В этой зоне лежат только `TMP_` / `PROXY_` материалы и scene-wrapper'ы, которые разрешено потом заменить без рефакторинга gameplay-кода.

## Правила

- Любой внешний ассет сначала проходит через `assets/intake/`, а не попадает сюда напрямую.
- Любой временный внешний ассет попадает только сюда, а не прямо в `features/`.
- Имена файлов и wrapper-сцен должны начинаться с `PROXY_` или `TMP_`.
- Для каждого используемого proxy-ассета должна существовать запись в `PROXY_asset_registry.csv`.
- Для каждого используемого proxy-ассета должна быть указана `magic_role`, если ассет участвует в активном magical vertical slice.
- Сцены игры должны ссылаться на wrapper/instance слой из этой папки, а не на исходный файл внешнего ассета напрямую.
- Замена ассета моделлером должна происходить точечно: сохранить `asset_id` и wrapper-path, а внутреннее содержимое сцены можно заменить.
