# Visual Bible

## Экран и scale policy

- Основной target: desktop `1920x1080`, `16:9`.
- Визуальный baseline для сцены и UI: фиксированная композиция под `16:9` с integer-oriented scale policy.
- Рабочая логика сетки:
  - мир: `16x16` пиксельный tile/grid rhythm
  - крупные объекты: кратно `32x32`
  - UI spacing: шаг `8px`

## Камера и читаемость

- Камера должна держать игрока в тактически читаемом центре кадра.
- Экранный шум недопустим: силуэты врагов, интерактива и контейнеров должны отличаться даже на placeholder-этапе.
- Любой новый визуальный элемент сначала проверяется на silhouette readability, затем на детализацию.

## Цвет и атмосфера

- Базовая среда: холодные асфальтово-зеленые и стальные диапазоны.
- Акценты:
  - токсичный зеленый для радиации/аномалий/опасности среды
  - теплый песочно-желтый для rare loot и важных UI-подсказок
  - приглушенный красный для урона и emergency feedback
- Сильные кислотные цвета вне сигналов опасности не использовать.

## Свет, туман, погода

- Ранний visual contract обязан учитывать:
  - направленный туман и атмосферную дымку
  - локальные пятна света
  - читаемые переходы день/пасмурно/грязный сумрак
- Освещение не должно разрушать пиксельную читаемость HUD и интерактива.

## UI Shell

- Плотность интерфейса должна быть близкой к `ZERO Sievert`: утилитарные панели, минимум декора, высокая информационная насыщенность.
- Каркас UI:
  - raid HUD
  - inventory/stash grid
  - trader screen
  - quest/task panel
  - bunker utility panels
- UI-панели строить на прямоугольных блоках с `1px`-логикой рамки и кратными `8px` отступами.

## Placeholder Standard

- Placeholder'ы рисуются сразу в финальном масштабе, не как абстрактные серые коробки.
- Для каждого placeholder'а обязательны:
  - итоговый footprint
  - различимый силуэт
  - цветовая роль
  - понятная принадлежность к фракции/классу объекта
